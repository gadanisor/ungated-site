This update introduces **GitHub project publishing**, improvements to the **writing assistant**, and several workflow enhancements for worldbuilding.

The goal of this iteration is to expand how projects can be shared and improve the writing experience inside the editor.

## Highlights

- Publish projects directly to **GitHub**
- Clone remote repositories from the Projects panel
- Configurable **Textlint** rules in Settings
- Editor **Quick Fix actions** for writing suggestions
- New **Item entity type** for worldbuilding
- Improved cloud synchronization for project metadata

## GitHub project publishing

Projects can now be published directly to GitHub from the **Source Control panel**.

Publishing enables:

- accessing projects from multiple machines
- collaborating with others
- using GitHub as a free remote backup

Once a repository is initialized and at least one commit exists, a **Publish on GitHub** option becomes available.

## Writing assistant improvements

The built-in writing assistant has been expanded with better configuration and editor integration.

### Configurable Textlint rules

Textlint can now be configured directly from the application settings.

You can enable or disable individual rules including:

- `textlint-rule-write-good`
- `textlint-rule-rousseau`
- `textlint-rule-no-unmatched-pair`
- `textlint-rule-no-start-duplicated-conjunction`

This allows writers to adjust linting behavior depending on their writing style or project needs.

### Editor Quick Fixes

Suggestions provided by Textlint can now be applied directly from the editor.

When a rule suggests a correction, the editor will display a **Quick Fix action** allowing the suggestion to be applied with a single click.

This makes it easier to correct style or grammar issues without interrupting writing flow.

## New entity type: Item

A new entity type called **Item** has been added.

Items allow writers to define important objects in their worlds, such as:

- artifacts
- magical objects
- tools
- weapons
- story-critical items

This expands the worldbuilding system beyond characters and locations.

## Other changes

- Deleting a project now removes its cloud record as well
- Projects can now be cloned from remote repositories
- Account deletion has been implemented
- Removed Pro feature notices from structured entity views

## Fixes

- Fixed synchronization issues affecting project avatars and metadata
